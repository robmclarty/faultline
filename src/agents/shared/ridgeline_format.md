# Ridgeline Output Format

The final output should be compatible with ridgeline's build specification
format. This includes:

- `overview.md` — Product overview and value proposition
- `spec.md` — Feature specifications with acceptance criteria
- `architecture.md` — Abstract architecture description
- `constraints.md` — Technical constraints and invariants
- `taste.md` — Coding style preferences

Each file should be self-contained markdown, readable without the others.
